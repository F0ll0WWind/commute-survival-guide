# 通勤生存指南 — 使用教程

### 第一步：环境准备

1. 注册高德开放平台账号：https://lbs.amap.com/
2. 创建应用，选择「Web 服务」类型，获取 API Key
3. 设置环境变量：
   ```bash
   export AMAP_API_KEY=你的API密钥
   ```
4. 安装 Python qrcode 库（扫码功能需要）：
   ```bash
   pip install qrcode
   ```

### 第二步：基础通勤查询

直接用自然语言告诉我起点和终点：

**示例对话：**

> 用户：我从深圳大学粤海校区到深圳大学丽湖校区，今天怎么走？

Skill 会自动：
1. 解析两个地址的坐标
2. 并行查询驾车、公交、骑行、步行四种方案
3. 获取当前天气
4. 综合分析后给出推荐

**输出示例：**

```
📍 深圳大学粤海校区 → 深圳大学丽湖校区

🚗 驾车  44分钟  ✅ 推荐
   距离 14.9km | 过路费 0 元 | 全程畅通

🚇 地铁  72分钟
   13号线 → 5号线 → 7号线 | 4.0元

🚲 骑行  83分钟  ❌ 不推荐
   当前小雨，路面湿滑

💡 建议驾车，44分钟到达，全程畅通。
```

### 第三步：带车牌号查询

如果你开车，可以提供车牌号，Skill 会自动判断限行：

> 用户：我从深圳湾一号到深圳大学，车牌粤B88888

Skill 会额外检查限行信息，如果今天限行会自动排除驾车方案。

### 第四步：指定偏好

你可以告诉我你的出行偏好：

> 用户：不想坐地铁，看看开车和骑车哪个快

> 用户：从家到公司有没有顺路的咖啡店？

### 第五步：扫码导航

Skill 会在推荐方案旁生成二维码：

> 用户：帮我查通勤路线，推荐方案生成二维码

输出中会包含一个在终端内直接显示的二维码，用手机高德 App 扫码后直接开始导航。

**注意：** 只有驾车/骑行/步行会生成二维码，公交/地铁不生成（因为不需要导航）。

### 第六步：查看通勤报告

使用一段时间后，可以查看月度统计：

> 用户：帮我统计一下这个月的通勤情况

Skill 会从本地记录中聚合数据，生成通勤月报，包括：
- 总通勤次数和平均耗时
- 最快/最慢一次通勤
- 最常走的路线
- 最佳出门时间
- 交通方式分布

## 常见问题

**Q: 为什么骑行方案显示"不推荐"？**
A: 当前有雨、雪、大风或极端温度时，Skill 会自动排除骑行和步行方案。

**Q: 限行信息准确吗？**
A: 限行数据来自高德 API，覆盖全国主要城市的限行政策。但建议以当地最新通告为准。

**Q: 通勤月报的数据保存在哪里？**
A: 保存在本地文件 `commute_log.json` 中，不会上传到任何云端。你可以随时删除。

**Q: 可以查询跨城市的通勤吗？**
A: 公交规划不支持跨城市，但驾车/骑行/步行可以。

**Q: 扫码导航需要安装什么？**
A: 需要安装 Python 的 `qrcode` 库（`pip install qrcode`），以及手机上安装高德地图 App。
